Problem Setting

C-Town Public Utility (CPU) is the main water distribution system operator of C-Town (Fig. 1). For many years, CPU has operated a fixed distribution network topology. In the last year, CPU has introduced novel smart technology to enable remote data collection from sensors in the field, and remote control of actuators. Several months after that new technology has been introduced, a major overflow in Tank T1 occurred. Shortly after, anomalous low levels in Tank T5 were noticed. While CPU personnel at SCADA were able see the anomalous readings for the latter episode, Tank T1 overflow took place unexpectedly while the T1 water level readings were always below the alarm thresholds. Searching for the causes, CPU engineers suspect potential cyber-attacks for both episodes. In particular, they are considering adversary profiles that are able to activate and deactivate the actuators in C-Town, as well as altering the readings of the sensors deployed in the network and interfering with the connections established between networked components.

Development data

CPU will provide the participants with the following data.

  • C-Town model in EPANET input format. Short description of C -Town with PLCs?
  • Historical data for the last two years before the smart devices were introduced.
    • The data is taken hourly, and provided in monthly intervals
    • The first two years of data are guaranteed to be without attacks
    • This data can be used to derive parameters related to normal system operations
  • Historical data for several months since the smart devices were introduced
    • The data is taken hourly, and provided in monthly intervals
    • The data is not guaranteed to be without attacks
  • Historical data for the last month, which includes the suspected attack that caused a tank overflow
